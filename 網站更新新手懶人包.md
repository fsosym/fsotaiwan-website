# 藝享愛樂網站更新新手懶人包

這份文件是給不熟悉程式的人使用的。只要照著步驟做，就可以更新網站內容、圖片，並把修改後的網站發布到線上。

---

## 一、更新網站前先確認

每次修改網站前，請先打開這兩個工具：

1. **Visual Studio Code**  
   用來修改網站文字、圖片路徑、網頁內容。

2. **GitHub Desktop**  
   用來把修改後的網站送到 GitHub，讓 Cloudflare 自動更新線上網站。

網站資料夾位置通常是：

```text
D:\fsotaiwanwebsite\fsotaiwan-website
```

請確認 VS Code 打開的是這個資料夾。

---

## 二、網站常用資料夾在哪裡？

網站主要檔案會放在：

```text
fsotaiwan-website
```

常見檔案與用途：

```text
index.html              首頁
news.html               最新消息
members.html            成員介紹
faculty.html            師資團隊
conductors.html         指揮團隊
activity-records.html   歷年活動
music-competition.html  藝享盃音樂大賽
```

圖片通常放在：

```text
assets\images
```

如果是新增老師照片、活動照片、演出照片，建議都放在 `assets\images` 裡面。

---

## 三、如何上傳圖片到指定位置？

### 步驟 1：先準備圖片

請先把圖片檔名改成英文或數字，不要使用中文、空格或特殊符號。

建議檔名：

```text
teacher-wang.jpg
concert-2026-01.jpg
violin-class-01.jpg
```

不要使用：

```text
王老師照片.jpg
2026 音樂會 照片.jpg
```

這樣比較不容易出錯。

---

### 步驟 2：把圖片放進資料夾

在 VS Code 左邊找到：

```text
assets
  images
```

把圖片拖進 `images` 資料夾。

例如你放進去的是：

```text
assets\images\teacher-wang.jpg
```

那網站裡要使用的圖片路徑就是：

```html
assets/images/teacher-wang.jpg
```

注意：網站裡的路徑要用 `/`，不是 Windows 的 `\`。

---

### 步驟 3：在網頁裡使用圖片

找到你要修改的 `.html` 檔案，例如：

```text
faculty.html
```

圖片通常會長這樣：

```html
<img src="assets/images/teacher-old.jpg" alt="老師姓名">
```

把 `src` 裡面的圖片路徑改成新圖片：

```html
<img src="assets/images/teacher-wang.jpg" alt="王老師">
```

改完後按：

```text
Ctrl + S
```

Live Server 會自動更新畫面。

---

## 四、如何修改網站文字？

### 步驟 1：找到要修改的頁面

例如：

- 首頁文字：打開 `index.html`
- 最新消息：打開 `news.html`
- 師資介紹：打開 `faculty.html` 或老師個人頁
- 指揮介紹：打開 `conductors.html` 或指揮個人頁

---

### 步驟 2：用搜尋快速找到文字

在 VS Code 裡按：

```text
Ctrl + F
```

輸入你看到的文字，例如：

```text
想加入藝享愛樂嗎？
```

找到後直接修改。

---

### 步驟 3：修改文字時注意

請只改中文字，不要亂刪 `<`、`>`、`</div>` 這類符號。

例如這段：

```html
<h2>想加入藝享愛樂嗎？</h2>
<p>歡迎與我們聯絡，了解更多課程與樂團資訊。</p>
```

可以改成：

```html
<h2>想了解藝享愛樂嗎？</h2>
<p>歡迎與我們聯絡，我們會協助您了解適合的學習方向。</p>
```

但不要刪掉：

```html
<h2>
</h2>
<p>
</p>
```

這些是網頁排版需要的標記。

---

## 五、如何新增一則最新消息？

打開：

```text
news.html
```

找到其他消息卡片，通常會像這樣：

```html
<article class="news-card">
  <p class="news-date">2026.07.01</p>
  <h3>活動標題</h3>
  <p>活動簡短說明文字。</p>
</article>
```

複製一整段，貼到你想放的位置，再修改日期、標題與內容。

例如：

```html
<article class="news-card">
  <p class="news-date">2026.08.15</p>
  <h3>夏季音樂會公告</h3>
  <p>藝享愛樂夏季音樂會即將登場，歡迎家長與朋友蒞臨聆賞。</p>
</article>
```

改完後按：

```text
Ctrl + S
```

## 六、如何修改老師或成員資料？

老師頁面可能會有很多獨立檔案，例如：

```text
violin-teacher-01.html
violin-teacher-02.html
conductor-liu.html
```

如果不知道是哪個檔案，可以用 VS Code 搜尋老師名字：

```text
Ctrl + Shift + F
```

輸入老師姓名，例如：

```text
劉嘉倫
```

VS Code 會列出含有這個名字的檔案，點進去就可以修改。

---

## 七、如何預覽修改後的網站？

在 VS Code 打開：

```text
index.html
```

按右鍵，選：

```text
Open with Live Server
```

瀏覽器會打開本機預覽網址。

修改檔案後，只要按：

```text
Ctrl + S
```

瀏覽器通常會自動更新。

如果沒有更新，請手動重新整理瀏覽器。

---

## 八、如何把修改發布到線上？

確認本機預覽正常後，打開 **GitHub Desktop**。

### 步驟 1：看 Changes

GitHub Desktop 左邊會列出你修改過的檔案。

---

### 步驟 2：填寫 Summary

左下角 Summary 可以寫這次改了什麼。

例如：

```text
更新最新消息
```

或：

```text
更新師資照片與介紹
```

---

### 步驟 3：Commit

按：

```text
Commit to main
```

---

### 步驟 4：Push

Commit 完後，按上方的：

```text
Push origin
```

---

### 步驟 5：等待 Cloudflare 更新

Push 後，Cloudflare 會自動更新網站。

通常等待：

```text
1～3 分鐘
```

再打開公開網址檢查：

```text
https://fsotaiwan.fantasy-fso-sym.workers.dev/
```

## 九、更新前後的安全習慣

### 1. 每次只改一小段

不要一次改太多頁。  
建議一次只改：

- 一則最新消息
- 一位老師資料
- 一個頁面區塊

這樣如果出錯，會比較容易找原因。

---

### 2. 改完一定要預覽

不要改完就直接 Push。  
請先用 Live Server 看過一次。

---

### 3. 不確定不要刪程式符號

看到這些符號請小心：

```html
<div>
</div>
<section>
</section>
<a href="...">
<img src="...">
```

如果只是改文字，通常只要改標籤中間的中文字即可。

## 十、常見問題

### Q1：圖片沒有出現怎麼辦？

請檢查三件事：

1. 圖片是否真的放在 `assets\images`
2. 檔名是否打錯
3. 網頁路徑是否使用 `/`

正確例子：

```html
<img src="assets/images/concert.jpg" alt="音樂會照片">
```

---

### Q2：改了文字但網站沒有變？

請確認：

1. 有沒有按 `Ctrl + S`
2. 瀏覽器有沒有重新整理
3. 是否修改到正確的檔案
4. 是否已經 Commit / Push 到 GitHub
5. Cloudflare 是否已經等 1～3 分鐘更新完成

---

### Q3：線上網站壞掉怎麼辦？

先不要慌。

可以做這幾件事：

1. 回 GitHub Desktop 看這次改了哪些檔案
2. 檢查是不是刪到 HTML 標籤
3. 如果真的找不到問題，可以回報給維護者，請他從備份或 GitHub 紀錄還原

---

## 十一、最簡單的更新流程總結

每次更新網站，只要記住這個流程：

```text
1. VS Code 修改文字或圖片
2. Ctrl + S 存檔
3. Live Server 預覽
4. GitHub Desktop Commit
5. Push origin
6. 等 Cloudflare 更新
7. 打開公開網站確認
```
